Getting Started
  1. Install: .NET 8 SDK, PostgreSQL, Elasticsearch 7.x, (optional: Docker)
  2. Clone: git clone https://github.com/Happy-Kumar-Sharma/GeoWorldHotelSearch.git
  3. Configure: Edit appsettings.json for DB and Elasticsearch
  4. Database: dotnet ef database update
  5. Run: dotnet run and open https://localhost:7034

Environment Variables & Configuration
  • appsettings.json - Main config file. Set your PostgreSQL connection string and Elasticsearch URL here.
  • Environment Variables:
    • ASPNETCORE_ENVIRONMENT (Development/Production)
    • ConnectionStrings__DefaultConnection (overrides DB connection)
    • Elasticsearch__Uri (overrides ES URL)
  • For secrets, use dotnet user-secrets in development.
Troubleshooting & Common Issues
  • PostgreSQL connection errors: Check your connection string and ensure the DB is running.
  • Elasticsearch not reachable: Make sure ES is running at the configured URL.
  • Port conflicts: Change ports in appsettings.json or Docker scripts.
  • Docker issues: Try docker-compose down and docker-compose up -d again.
  • Migration errors: Delete the DB and rerun dotnet ef database update.
  • Seeding fails: Check logs for details; ensure DB and ES are up.
  • UI not updating: Use dotnet watch run for hot reload.
  • Still stuck? Check logs, open an issue on GitHub, or see the FAQ below.
Deployment
  • Production Build: dotnet publish -c Release
  • Deploy to Linux server: Copy published files, set up Nginx/Apache reverse proxy, and run as a systemd service.
  • Docker Compose: Use docker-compose up -d for DB and ES. Add your own Dockerfile for the app if desired.
  • Cloud: Deploy to Azure App Service, AWS Elastic Beanstalk, or DigitalOcean App Platform. Set environment variables as needed.
  • HTTPS: Use a reverse proxy (Nginx/Apache) or configure Kestrel for SSL.

Security Best Practices
  • Never commit secrets to source control. Use environment variables or dotnet user-secrets.
  • Keep dependencies up to date. Run dotnet list package --outdated.
  • Use HTTPS in production.
  • Restrict database/network access to trusted sources.

Performance & Scaling Tips
  • Use pagination for all hotel/search queries.
  • Elasticsearch is optimized for millions of records, but tune JVM heap for large datasets.
  • Use connection pooling for PostgreSQL.
  • Deploy behind a reverse proxy for better performance and security.
  • Monitor logs and metrics (see admin dashboard for analytics).

Rolling Featured Hotels
  • Mark/Unmark as Featured: Admins and users can mark any hotel as Featured directly from the hotel list, details, or edit screens.
  • Home Page Display: Only featured hotels are shown in the "Featured Hotels" section on the home page. If none are featured, a message is shown.
  • How to Use:
    • On the Hotels list or details page, click the star button to mark/unmark a hotel as featured.
    • On the Edit Hotel page, use the "Mark as Featured Hotel" checkbox.
    • Changes are reflected instantly in the UI and on the home page.
  • API: The backend exposes a SetFeatured action for toggling featured status.
  • Database: The Hotels table includes an IsFeatured column (bool).
